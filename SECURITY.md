# Security Policy

## Supported Versions

Because Pulsera is currently in its initial release phase, only the latest major version is officially supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is a fundamental priority for Pulsera. As this is a solo-developed open-source project, community help in identifying and reporting vulnerabilities is deeply appreciated.

If you discover a security vulnerability within the Pulsera application or its backend architecture, please **do not** disclose it publicly by creating a GitHub issue. Instead, please follow this responsible disclosure process:

1. **Report:** Send an email directly to [mohamedyehiaali4@gmail.com] with the subject line `[Security Vulnerability] Pulsera`.
2. **Details:** Include a detailed description of the vulnerability, the specific components affected (e.g., TOTP generation, Firestore rules, or Flutter frontend), steps to reproduce the issue, and the potential impact.
3. **Response Expectation:** You can expect an initial acknowledgment of your report within 48 to 72 hours. 
4. **Resolution:** I will evaluate the report and, if accepted, work on a patch as quickly as possible. You will be kept updated on the progress. Once the fix is merged, a new security release will be published.

Thank you for your responsible disclosure and for helping keep Pulsera secure!
